quickstart · duckdb-first · no local docker daemon required

From install to a running pipeline in a few commands.

This is the recommended first-run path for flowctl. It creates a standard flowctl/v1 pipeline, starts an embedded control plane, and writes contract events to a local DuckDB file.

01 install
go install github.com/withobsrvr/flowctl@latest
export PATH="$HOME/go/bin:$PATH"
flowctl version

Alternative: curl -sSL https://flowctl.withobsrvr.com/install.sh | sh

02 generate
flowctl init --preset testnet-duckdb

Equivalent explicit form:

flowctl init --non-interactive --network testnet --destination duckdb
03 validate
flowctl validate stellar-pipeline.yaml
04 run
flowctl run stellar-pipeline.yaml

On first run, flowctl resolves and downloads components, starts the control plane, launches components, and wires the pipeline.

05 inspect
# in another terminal
flowctl status
flowctl pipelines active
06 query
duckdb stellar-pipeline.duckdb "SELECT event_type, COUNT(*) FROM contract_events GROUP BY event_type"

Stop the pipeline first if you want exclusive access to the DuckDB file from the CLI.

troubleshooting

Port collision

flowctl run --control-plane-port 0 stellar-pipeline.yaml

Component download problem

flowctl run stellar-pipeline.yaml --log-level=debug

Status cannot connect

flowctl status --control-plane-address 127.0.0.1 --control-plane-port 9090

No DuckDB CLI installed

That only affects ad-hoc querying from your shell. The pipeline can still write the DuckDB file successfully.
